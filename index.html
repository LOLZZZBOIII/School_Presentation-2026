What is Git?

  • Distributed version control system
  • Tracks changes to files, enables collaboration
  • Local commits & history

What is GitHub?

  • Hosting for git repositories
  • Provides collaboration tools: pull requests, issues, actions, pages
  • Web UI for viewing code, commits, and history

Common workflow

  1. Clone repository to local machine
  2. Create a branch for your change
  3. Make edits and commit locally
  4. Push branch and open a Pull Request (PR)
  5. Get review, then merge

Essential Git Commands

git clone <repo-url>
git checkout -b my-feature
git add .
git commit -m "Brief message"
git push -u origin my-feature

Using GitHub

  • Create repo (or fork)
  • Open issues to discuss bugs/ideas
  • Use Pull Requests for code review
  • Use Actions for CI and Pages for hosting

Publishing with GitHub Pages

  1. Push the site files (index.html, styles.css, script.js) to the default branch (main)
  2. Go to your repo Settings → Pages
  3. Select "Branch: main" and "Folder: / (root)" then Save
  4. Your site becomes: https://LOLZZZBOIII.github.io/School_Presentation-2026/

Example: https://LOLZZZBOIII.github.io/School_Presentation-2026/

Tips

  • Keep commit messages short and clear
  • Use branches for separate features
  • Write README with usage instructions
  • Protect main branch in repo Settings if working with a team
